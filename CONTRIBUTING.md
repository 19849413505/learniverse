# Contributing Guidelines

Welcome to the Learniverse project! We appreciate your interest in contributing. Below are the guidelines for developers:

## Contribution Guidelines  
1. **Fork the repo**: Create a personal copy of the repository by forking it.  
2. **Create a branch**: Use descriptive branch names. It's recommended to use a format like `feature/your-feature-name` or `bugfix/issue-name`.  
3. **Make your changes**: Implement your feature or fix.  
4. **Commit your changes**: Write clear, concise commit messages.  
5. **Push your changes**: Push your changes to your forked repository.  
6. **Create a Pull Request**: Describe your changes and submit your pull request for review.  

## Guidelines de Contribution  
1. **Forkez le dépôt**: Créez une copie personnelle du dépôt en le forkant.  
2. **Créez une branche**: Utilisez des noms de branche descriptifs. Il est recommandé d'utiliser un format comme `feature/votre-nom-de-fonctionnalité` ou `bugfix/nom-du-bug`.  
3. **Effectuez vos modifications**: Implémentez votre fonctionnalité ou correction.  
4. **Commitez vos modifications**: Écrivez des messages de commit clairs et concis.  
5. **Poussez vos modifications**: Poussez vos modifications dans votre dépôt forké.  
6. **Créez une Pull Request**: Décrivez vos modifications et soumettez votre pull request pour examen.

Thank you for contributing!  
Merci pour votre contribution!
